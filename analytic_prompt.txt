### Endpoint 1: Program Overview
GET /analytics/program-overview?org_id=

Purpose: Top-level org dashboard — how many programs, cohorts, participants across statuses.

DTO to add (dto.go):
```go
type ProgramOverviewResponse struct {
    TotalPrograms    int `json:"total_programs"`
    ActivePrograms   int `json:"active_programs"`
    DraftPrograms    int `json:"draft_programs"`
    DeliveredPrograms int `json:"delivered_programs"`

    TotalParticipants int `json:"total_participants"`
    AtRiskCount      int `json:"at_risk_count"`
    AvgCompletion    float64 `json:"avg_completion"`
}
```

SQL logic:
- programs table → GROUP BY status for program counts WHERE org_id = $1
- cohorts → COUNT WHERE program_id IN (programs for this org)
- enrollments → COUNT participants, AVG(completion_percent), COUNT WHERE risk_level = 'high'

### Endpoint 2: Cohort Progress (Participant-level health table)
GET /analytics/cohort-progress?cohort_id=

Purpose: Per-participant row showing their completion, risk level, sessions attended, submissions graded.
This is the "cohort health table" the PM uses most.

DTO:
```go
type ParticipantProgress struct {
    UserID             string  `json:"user_id"`
    Name               string  `json:"name"`
    Email              string  `json:"email"`
    CompletionPercent  float64 `json:"completion_percent"`
    RiskLevel          string  `json:"risk_level"`   // "low" | "medium" | "high"
    EnrollmentStatus   string  `json:"enrollment_status"`
    SessionsAttended   int     `json:"sessions_attended"`
    TotalSessions      int     `json:"total_sessions"`
    SubmissionsGraded  int     `json:"submissions_graded"`
    TotalSubmissions   int     `json:"total_submissions"`
    LastActive         *time.Time `json:"last_active"` // from audit_logs WHERE action='login'
}

type CohortProgressResponse struct {
    CohortID     string               `json:"cohort_id"`
    Participants []ParticipantProgress `json:"participants"`
    Summary struct {
        TotalEnrolled int     `json:"total_enrolled"`
        AtRiskCount   int     `json:"at_risk_count"`
        AvgCompletion float64 `json:"avg_completion"`
    } `json:"summary"`
}
```

SQL logic:
- JOIN enrollments → users → session_attendance → submissions
- sessions_attended: COUNT(sa) WHERE sa.status = 'present' AND sa.session_id IN (SELECT id FROM class_sessions WHERE cohort_id=$1)
- last_active: SELECT MAX(created_at) FROM audit_logs WHERE user_id = u.id AND action = 'login'
- ORDER BY completion_percent DESC

### Endpoint 3: Activity Completion Breakdown
GET /analytics/activity-completion?cohort_id=

Purpose: Per-activity row showing how many participants completed it, average score, overdue count.

DTO:
```go
type ActivityCompletionRow struct {
    ActivityID      string  `json:"activity_id"`
    Title           string  `json:"title"`
    ActivityType    string  `json:"activity_type"`
    TotalParticipants int   `json:"total_participants"`
    CompletedCount  int     `json:"completed_count"`
    CompletionPct   float64 `json:"completion_pct"`
    AvgScore        *float64 `json:"avg_score"` // null if no submissions
    OverdueCount    int     `json:"overdue_count"`
    PhaseName       string  `json:"phase_name"`
}

type ActivityCompletionResponse struct {
    CohortID   string                  `json:"cohort_id"`
    Activities []ActivityCompletionRow `json:"activities"`
}
```

SQL logic:
- activities JOIN phases (for phase name), LEFT JOIN activity_progress, LEFT JOIN submissions
- completed_count: COUNT(ap) WHERE ap.status = 'completed'
- overdue: WHERE due_date < NOW() AND ap.status != 'completed' (verify due_date column exists on activities or activity_progress)
- avg_score from submissions WHERE status = 'graded'
- Filter: activities WHERE phase_id IN (SELECT id FROM phases WHERE program_id = (SELECT program_id FROM cohorts WHERE id=$1))
  AND cohort_id matched via enrollments

### Endpoint 4: Attendance Heatmap
GET /analytics/attendance-heatmap?cohort_id=

Purpose: Per-session attendance breakdown — present / absent / late counts and rate.

DTO:
```go
type SessionAttendanceRow struct {
    SessionID     string     `json:"session_id"`
    Title         string     `json:"title"`
    ScheduledAt   time.Time  `json:"scheduled_at"`
    TotalExpected int        `json:"total_expected"`
    PresentCount  int        `json:"present_count"`
    AbsentCount   int        `json:"absent_count"`

    AttendanceRate float64   `json:"attendance_rate"`

```

SQL logic:
- class_sessions WHERE cohort_id = $1
- LEFT JOIN session_attendance → COUNT by status
- attendance_rate = present_count / total_expected * 100
- ORDER BY scheduled_at ASC

### Endpoint 5: Submission Grade Distribution
GET /analytics/submission-grades?cohort_id=

Purpose: Grade distribution per gradable activity. Used for the bar chart (buckets: 0-49, 50-59, 60-69, 70-79, 80-89, 90-100) plus avg and pending count.

DTO:
```go
type GradeBucket struct {
    Label string `json:"label"`  // "0-49", "50-59", etc.
    Count int    `json:"count"`
}

type ActivityGradeStats struct {
    ActivityID   string        `json:"activity_id"`
    Title        string        `json:"title"`
    AvgGrade     float64       `json:"avg_grade"`
    PendingCount int           `json:"pending_count"`
    GradedCount  int           `json:"graded_count"`
    Buckets      []GradeBucket `json:"buckets"`
}

type SubmissionGradesResponse struct {
    CohortID   string               `json:"cohort_id"`
    Activities []ActivityGradeStats `json:"activities"`
}
```

SQL logic:
- submissions WHERE activity_id IN (activities for this cohort's program)
- pending_count: COUNT WHERE status = 'submitted' (not yet graded)
- graded_count: COUNT WHERE status = 'graded'
- Buckets: use CASE WHEN grade < 50 THEN '0-49' WHEN grade < 60 THEN '50-59' etc. GROUP BY bucket
- avg_grade: AVG(grade) WHERE status = 'graded'

### Endpoint 6: Session Summary
GET /analytics/session-summary?cohort_id=

Purpose: High-level session stats — how many sessions delivered, total hours, action items status.

DTO:
```go
type SessionSummaryResponse struct {
    CohortID          string  `json:"cohort_id"`
    TotalScheduled    int     `json:"total_scheduled"`
    TotalDelivered    int     `json:"total_delivered"`
    TotalHours        float64 `json:"total_hours"`
    AvgDurationMins   float64 `json:"avg_duration_mins"`
    ActionItemsOpen   int     `json:"action_items_open"`
    ActionItemsClosed int     `json:"action_items_closed"`
    ActionItemsOverdue int    `json:"action_items_overdue"`
    PollParticipationRate float64 `json:"poll_participation_rate"`
}
```

SQL logic:
- class_sessions WHERE cohort_id=$1
- delivered: WHERE status = 'completed' (verify status value in migrations)
- total_hours: SUM(duration_mins)/60
- action items: JOIN session_action_items → GROUP BY status
- overdue: WHERE due_date < NOW() AND status = 'open'
- poll rate: COUNT(session_poll_votes) / COUNT(expected participants) × 100
